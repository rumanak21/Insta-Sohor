Q-1: How does JavaScript works?

Answer: JavaScript is a high-level and single-threaded programming language. It is also garbage-collected, interpreted, prototype-based, and dynamic language with a non-blocking event loop. It works in Chrome and Node.js using Google's V8 engine. JavaScript is single-threaded, which means it has a single Call Stack. So it can do one thing at a time.

Q-2: What is the difference between local storage and session storage?

Answer: LocalStorage is a Storage object which can be used to access the current origin's local storage space. SessionStorage is a storage object which can be used to access the current origin's session storage space. Data of localStorage doesn't expire but, data of sessionStorage is cleared when the page session ends. LocalStorage can save up to 10MB, SessionStorage will delete after closing your browser, also can save less than LocalStorage, like up to 5MB. LocalStorage is not session-based, needs to be deleted via JS or manually. SessionStorage session-based and working per window or tab.
